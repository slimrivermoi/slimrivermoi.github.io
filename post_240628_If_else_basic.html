Treasure Island Game

A simple way to illustrate If-Else logic in Python

Looking back at my earliest Python coding lessons, understanding if-else logic is perhaps one of the most important foundation one should learn.

Let's say we are building a Treasure Island Game where the player has to go through a series of challenges to find the treasure. Life is full of choices, and this game is no exception! In our game, you will be asked to make choices and when the wrong one is chosen, your game is over.

To start, let's draw our game logic:

Step 1: draw a flowchart

Drawing a flowchart can be really helpful before you start coding. There are many drawing tools out there, I personally like the draw.io for this purpose.

As you can see in the chart above, we are creating 3 challenges (or choices) for the player to decide. Every step along the way, if the player chooses the wrong option, the game will be over. Based on the logic above, we will be building a "nested if" statements in our code, starting from Choice1, then Choice2 and Choice3.

Step 2: code the "if-else" for Choice 1, then for Choice 2 and Choice 3.

In Choice 1, if player selected "left" they get to continue with the game. The game will be over if they choose otherwise.

Common Mistakes

  • Indentation: this is a common mistake people often made. Be sure to use "tab" or "space" to indent your next line of code.
  • Confusion between when to use "if-else" and "elif": Whenever in doubt, think about the decision you want Python to do: do you want Python to check all conditions and perform all tasks?

Here is a simple illustration on how I would differentiate the two statements.

If-Else statement: check each IF condition and perform the actions if conditions are met. 
        
if condition_1 == True:
    perform action_a
    
if condition_2 == True:
    perform action_b

if condition_3 == True:
    perform action_c

If-Elif Statement: only check condition 1 first. If condition 1 is met, it will neither check condition 2 nor 3. 

        
if condition_1 == True:
    perform action_a

elif condition 2 == True:
    perform action_b 

elif condition 3 == True:
    perform action_c   

else:
    perform action_d

The final mistake I have made a lot is making my If-else logic too complicated (with too many nested If's, Elif, Else). If you find yourself feeling lost in your own If-logic, perhaps it is good to sit back, take a break and relook at the problem again.
